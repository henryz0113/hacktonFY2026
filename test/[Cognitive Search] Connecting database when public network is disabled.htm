[Cognitive Search] Connecting database when public network is disabled.

I am unable to add database from overview and as well as not unable to add data sources via menu Portal.

Are you seeing your Import Data disabled from the Portal after configuring your Cognitive Search to be privately accessed network? Are you unable to add database from the data sources from the Management tab? In this article you will find out how to connect the database of your desire in Cognitive Search that has disabled public network access.

 

I. What is next after disabling your public network in your Cognitive Search?

When you disable public network access in your Cognitive Search you will be able to explore through some services of Cognitive Search via Porta but, you will encounter some functionality being inaccessible such as adding Data Sources, where this article will explain you on how to resolve this issue.

 

II. How to set up database through the Portal.

If you are reading this article, that means your import data is unclickable just like the picture below. Also, when you hit the Data Sources, to add the data source, the add function is disabled as well.

This is because you have not configured the shared private link on your network side. Once you get that setup, you will be able to add your data sources on the Cognitive Search.

 

1. Go to Network > Shared Private Access.

Here you will have add the data source to have access to the Private Endpoint.

 

2. Add Share Private Access and fill out the pop up as below with the correct information.

Make sure that the Resource Type is to your desire. In this example, we are using Storage Account and Blob Container.

 

3. Make sure the Provisioning State has turned to Succeed as below.

 

4. Go to your Data Sources tab on the left and add your Storage Account that you have just configured.

Press Choose and existing connection where you will see the pop-up as below.

 

 

5. Once you choose the Storage Account and the Container, then make sure you add the Name of this Data Source. Make sure to SAVE it!

 

6. When you save it go have to the Cognitive Search Data Source tab and refresh it!

 

III. What are the limitations to the Private Shared Access?

 

Unfortunately, there are some limitations to using Private Shared Access. There are things to consider when using it, such as some resource types are still in Preview. Also, when you are using the Indexer, make sure you are in the same network environment of Cognitive Search.

 

These are the limitations explained in detail via our documentation.

 

Several of the resource types used in a shared private link are in preview. If you're connecting to a preview resource (Azure Database for MySQL, Azure Functions, or Azure SQL Managed Instance), use a preview version of the Management REST API to create the shared private link. These versions include 2020-08-01-preview or 2021-04-01-preview.

Indexer execution must use the private execution environment that's specific to your search service. Private endpoint connections aren't supported from the multi-tenant environment. The configuration setting for this requirement is covered in this article.

 

IV. Conclusion

I hope through this article, you are now able to add the Data Source to the Private Cognitive Search. Make sure you are aware of the limitations as well. Furthermore, not only can you add Data Source through Private Share Access, but you can also add other services such as Key Vault if needed. Make sure to check this link as well for more information.
